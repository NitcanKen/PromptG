## Required Decisions

1. Data remains in `data/app.db`.
2. Images remain in `data/uploads/`.
3. Mimo model selector is visible in UI.
4. Mimo default model is `mimo-v2.5-pro`.
5. Available models:
   - `mimo-v2.5-pro`
   - `mimo-v2.5`
   - `mimo-v2-pro`
   - `mimo-v2-omni`
6. `XIAOMI_MIMO_API_KEY` must come from `.env.local` or runtime environment.
7. Seed images must be generated during implementation by Codex's built-in image2 / GPT Image 2 capability. Do not use generic placeholders, stock images, or unlicensed web images.
8. All user-visible UI must be Traditional Chinese. Internal code and docs may be English. Prompt body content may remain English when it is actual reusable image Prompt text.
9. Seed atom titles, subtitles, tags, and notes must be Traditional Chinese; seed atom Prompt bodies may be English.
10. The product must not integrate image2 / GPT Image 2 as an app feature, API route, or runtime dependency. The user does not have and does not need an image2 API key.

## Mimo API Requirements

Base URL:

```bash
https://token-plan-sgp.xiaomimimo.com/v1
```

Environment variables:

```bash
XIAOMI_MIMO_API_KEY=...
XIAOMI_MIMO_BASE_URL=https://token-plan-sgp.xiaomimimo.com/v1
XIAOMI_MIMO_MODEL=mimo-v2.5-pro
```

Rules:

1. Never hardcode a real API key.
2. Never expose the API key to client components.
3. Frontend calls only local API routes.
4. Server route calls Mimo through an OpenAI-compatible client or direct HTTP implementation.
5. Validate model name from UI against the allowed list.
6. Validate Mimo output with Zod before returning it to the client.
7. If Mimo returns invalid JSON, attempt at most one repair request.
8. Parsed atoms are drafts only and must not be saved automatically.

## Prompt Parser Output Contract

The parser API must return:

```json
{
  "items": [
    {
      "category": "人設",
      "title": "短標題",
      "subtitle": "一句效果說明",
      "prompt": "可直接拼接進完整 Prompt 的片段",
      "negativePrompt": "",
      "tags": ["標籤一", "標籤二"],
      "notes": "拆解依據或使用建議"
    }
  ]
}
```

Constraints:

1. `category` must be one of the 16 fixed categories.
2. `title`, `subtitle`, and `prompt` must be editable in UI before save.
3. `tags` max 8.
4. Empty `negativePrompt` is valid.
5. Model uncertainty belongs in `notes`.

## Risks

1. Mimo API behavior may differ from OpenAI-compatible assumptions. Keep the client wrapper isolated.
2. Image generation may produce previews that are visually too similar. Regenerate weak seed assets rather than shipping indistinct cards.
3. Search and tag filtering can become slow if implemented with repeated full client scans. MVP can be simple, but keep the query boundaries clean.
4. Gallery snapshot shape must be versioned or at least tolerant of missing atoms.
